Brevity is the soul of wit

The way I see it, any piece of software should be devoid of bloat. From a UI point of view, this means as few visual components as necessary, all of which do their own job, nothing more, nothing less. But such a software should also be (at least somewhat) eye-pleasing.

From a developer's point of view, software should at least follow the well-known principles: DRY, KISS, etc. This also means using modern tech for development. For example, a web application created using Svelte makes for an app that comes with minimum JS, is simple enough to develop and, most importantly, tries to solve problems that persist at the current moment of time in the IT sphere. You can even use Svelte as an SSG!
